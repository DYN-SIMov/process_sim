Look at the current context (open file, recent changes, or the task I describe below).

Identify 1–3 specific, actionable suggestions for how I could use AI tools more effectively here. 
Focus on:
- Domain-specific opportunities: e.g., using AI to validate thermodynamic intuition, 
generate test cases for VLE models, or explore parameter sensitivity
- General workflow improvements: e.g., using AI for code review, documentation, or 
refactoring suggestions, testing

Consider the following when making suggestions:
- Make sure the suggestions are relevant to the current context and not generic advice. 
For example, if I’m working on a VLE model, suggest how AI could help validate it or generate 
test cases, rather than just saying “use AI for code review”.
- Make sure the suggestions are safe and reliable. I am developing a very complex and critical 
software, so I need to be careful problems such as hallucinations, injection, or over-reliance on 
AI tools. If there are risks, explain them and how to mitigate them.

Be concise and specific to the current context. Skip generic advice.
